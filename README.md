# VAE-CTABGAN

VAE-CTABGAN is a tabular data synthesis framework that integrates a Variational Autoencoder (VAE) into the CTAB-GAN pipeline.  
Instead of relying only on random noise, the model uses latent representations learned from continuous financial features to generate more structured and semantically meaningful synthetic tabular samples.

This repository contains the implementation of **data preprocessing**, **joint VAE-GAN training**, and **synthetic sample generation** for imbalanced tabular learning experiments.

## Usage
```bash
python main.py --mode {preprocess,train,generate,only_train}
```
The repository supports four execution modes:
- preprocess
- train
- generate
- only_train

Mode 1: Preprocess Only
Use this mode if you only want to preprocess the raw dataset and save the transformed outputs.
```bash
python main.py --mode preprocess
```
This step prepares the following artifacts: processed CSV file, transformer object (transformer.pkl), dataprep object (dataprep.pkl) 
These outputs are required before training or sample generation.

Mode 2: Train
Use this mode to train the VAE-CTABGAN model after preprocessing.
```bash
python main.py --mode train
```
Mode 3: Only Train
Use this mode when preprocessing has already been completed and you want to train using the saved processed artifacts.
```bash
python main.py --mode only_train
```
This mode assumes the following files already exist: preprocess/preprocessed.csv preprocess/transformer/transformer.pkl preprocess/dataprep/dataprep.pkl

Mode 4: Generate Synthetic Samples
Use this mode to generate synthetic samples from a trained model.
```bash
python main.py --mode generate
```

## Quick Start

A typical end-to-end workflow is as follows:

```bash
# 1. Preprocess the raw Lending Club dataset
python main.py --mode preprocess

# 2. Train VAE-CTABGAN using the preprocessed artifacts
python main.py --mode only_train

# 3. Generate synthetic minority-class samples from the trained model
python main.py --mode generate
```

Alternatively, preprocessing and training can be executed together:

```bash
python main.py --mode train
```

After generation, the synthetic samples can be used for downstream imbalanced learning experiments, such as minority-class credit-risk screening.

* `preprocessed.csv`: transformed tabular data used for model training
* `transformer.pkl`: fitted transformer for inverse transformation
* `dataprep.pkl`: preprocessing metadata and feature configuration

* `vae_ctabgan_epoch{epoch}.pth`: epoch-stamped final VAE-CTABGAN checkpoint
* `generated_samples.csv`: generated synthetic minority-class samples

The exact file paths and filenames can be adjusted in `main.py` and the data preparation scripts.

## Important Notes on Data Leakage

This project is designed for loan-origination-time modeling. Variables that are only observed after loan performance is realized should not be used as predictive input features.

Examples of excluded post-outcome variables include:

- `total_pymnt`
- `total_pymnt_inv`
- `last_fico_*`
- `debt_settlement_flag`
- `recoveries`
- `collection_recovery_fee`
- `last_pymnt_d`
- `out_prncp`
- `out_prncp_inv`
- variables directly derived from repayment outcomes

The repayment label `loan_status` is used only as the target label and semantic condition during generative training, not as an input feature for downstream credit-risk prediction.

## Downstream Use in the Paper

In the related study, synthetic samples generated by VAE-CTABGAN are used only to improve the training of the minority-class credit-risk screening classifier. They are not directly used to train the profit scoring model or to construct portfolios.

The overall decision-support pipeline is:

```text
VAE-CTABGAN synthetic minority generation
        ↓
Default-risk screening classifier
        ↓
Investment-candidate filtering
        ↓
Profit scoring on real candidate loans
        ↓
Top-k portfolio construction
```

## Dataset

This project uses the **Lending Club dataset (2007–2020)** for experiments on imbalanced learning and synthetic tabular data generation.

- Dataset source: Lending Club loan records
- Period: **2007 to 2020**
- **[Dataset link](https://www.kaggle.com/datasets/ethon0426/lending-club-20072020q1)**

### Data Description

<img src="https://github.com/user-attachments/assets/633da717-78dc-4ace-bf41-00cae8fd82f4" width="851" height="812" alt="Image"/>
The selected predictive features consist of 10 continuous variables and 5 categorical variables, chosen to capture key aspects of borrower creditworthiness, repayment capacity, and loan characteristics while avoiding post-origination outcome information.  
The continuous variables include financial and credit-related indicators such as FICO score, income, debt-to-income ratio, interest rate, installment amount, revolving credit usage, revolving balance, bankcard utilization, and credit history length.  
The categorical variables describe loan and borrower profile information available at origination, including loan subgrade, home ownership status, loan purpose, employment length, and loan term.  
The repayment label (`loan_status`) is encoded separately as the target label for semantic consistency during generative training and should not be treated as a predictive input feature for downstream credit scoring.

## Model Architecture
<img width="893" height="555" alt="Image" src="https://github.com/user-attachments/assets/5f70c447-82ae-4755-9d50-469400ed2ce9" />

The proposed VAE-CTABGAN framework consists of three main parts: **feature encoding**, **Stage 1 VAE warm-up learning**, and **Stage 2 adversarial VAE-CTABGAN learning**.  
First, raw tabular data are transformed into an encoded representation. Continuous and mixed variables are processed through mixture-based normalization, where each value is represented by a normalized scalar and its corresponding mode indicator, while categorical variables are converted into conditional vectors through log-frequency sampling.  

In **Stage 1**, the VAE encoder learns latent representations from the encoded continuous features. The latent vector is then concatenated with the conditional vector and passed to the generator, which reconstructs synthetic tabular samples. This warm-up stage helps the model learn a structured latent space before adversarial training begins.  

In **Stage 2**, the generated samples are further refined through adversarial learning. The discriminator evaluates whether samples are real or synthetic, the information loss compares real and synthetic discriminator feature statistics using a fresh real batch at each generator step, and the classifier enforces semantic consistency with the target repayment label. During sample generation, the model uses latent vectors produced by the trained VAE encoder from seed borrower feature representations rather than drawing the generator input from pure random noise. By combining latent representation learning, conditional generation, and adversarial optimization, VAE-CTABGAN aims to generate more realistic and semantically meaningful tabular data than noise-driven tabular GANs.

## Example pipeline code
The `example/` directory contains the decision process used in the paper. It shows how VAE-CTABGAN-generated minority default samples are used to strengthen the default-risk screening stage, and then how the screened candidate loans are passed to a profit scoring model for return-oriented portfolio selection.

The workflow follows the main experimental logic of the study:

```text
VAE-CTABGAN synthetic minority samples
        ↓
Default-risk screening classifier
        ↓
Screened real loan candidates
        ↓
Profit/return scoring model
        ↓
Top-k portfolio construction
```

Synthetic samples are used only to improve the risk-screening classifier. The profit scoring stage and portfolio evaluation are based on real loans, so the selected Top 10%, Top 20%, and Top 30% portfolios are constructed from screened real test observations rather than from generated samples.

The notebook is intended as a compact reference implementation of the proposed decision-support logic, not as a full reproduction package for every table and figure in the paper.
